## 二、重构原则

### 核心原则：行为驱动，而非数值驱动

1. **每个 BE 的主触发必须是玩家的选择/行动**——不是好感度数值
2. **好感度可以调节后果严重性，但不能是触发门控**——好感低不等于该死
3. **用"信任事件完成度"替代"好感度数值"作为二级门控**——你完成了多少信任事件，比数值更直观
4. **纯好感总和型 BE（BE19/BE22）必须重构为"孤立行为模式"或"信息隐瞒模式"**——全员离你而去是因为你选择了独来独往，不是因为一个数字
5. **玩家必须能从行为推断后果**——如果你选择当众顶撞白鹤姬，你应该能预见风险

### 三级门控架构

每个 BE 采用以下三级结构：

```
第一级：关键选择/行动（必须触发）
  → 玩家做出了某个特定选择或行为
第二级：信任事件完成度（调节容错）
  → 完成足够信任事件 → 获得挽回机会
  → 未完成 → 直接进入BE
第三级：好感度（仅影响后果表现）
  → 高好感：BE场景中有更多情感张力
  → 低好感：BE场景更冷酷
  → 不影响是否触发
```

## 三、逐条重构方案

### BE04「白鹤的囚笼」Day 10~12

**原触发**：白鹤姬#4香道中好感度太高且选择"顺从" → 被魅惑控制

**问题**：好感度高被"惩罚"，玩家投入感情反而触发坏结局

**重构后**：
- **第一级（主触发）**：在 Day 10~12 的白鹤姬#4~#5 对话中，连续三个关键节点均选择"顺从/不质疑"选项（三次顺从形成行为模式，非单次选择）
- **第二级（信任事件）**：若在 Day 1~9 期间完成了"雏鹃的镜子事件"或"神鸦的第一次占卜"（≥1个独立判断事件），则在第三次顺从前解锁"内心抗拒"子选项，选择后不触发BE04
- **第三级（好感度）**：不影响触发。好感高时BE04场景中雪代语气更温柔（"你终于属于我了"）；好感低时更冷酷（"果然只是条狗"）
- **关键改变**：好感度高不再是触发条件，连续顺从的行为模式才是

### BE05「过界的代价」Day 11~14

## 四、好感度系统定位调整

### 原定位（问题）
好感度 = BE 触发门控 + 结局判定条件 + 角色关系描写参考
→ 好感度承担了过多功能，玩家无法从数值推断后果

### 新定位
| 功能 | 原系统 | 新系统 |
|------|--------|--------|
| BE 触发门控 | 好感度阈值 | **行为选择 + 信任事件完成度** |
| 结局判定 | 好感度达标 | **关键选择 + 信任事件 + 证据链**（已在Day45/Day54判定系统中实现） |
| 角色关系描写 | 好感度参考 | 好感度参考（保留） |
| BE 场景氛围 | 无 | **好感度影响BE场景描写语气**（新增） |

### 好感度的新角色
好感度从此**只影响叙事表现层**（对话语气、场景氛围、情感张力），**不再影响逻辑判定层**（BE触发、结局分叉）。逻辑判定由"行为选择"+"信任事件完成度"承担。

---

## 五、信任事件完成度系统说明

### 定义
"信任事件完成度" = 玩家在特定角色好感事件链中，完成了多少个"展现独立判断/建立深层关系"的事件。

### 与好感度的区别
- **好感度**：通过对话选择累积的数值，玩家可以通过"讨好"选项快速堆高
- **信任事件完成度**：需要完成特定事件链中的关键节点，不能通过简单"讨好"获得

### 为什么用信任事件替代好感度
1. **可推断性**：玩家能回忆"我完成了哪些事件"，但无法精确知道"好感度是多少"
2. **不可刷分**：信任事件是有限的故事节点，不能重复刷
3. **叙事一致性**：角色信任你是因为你做了什么事，不是因为一个数字到了某个值

### 实现方式
在 Day45/Day54 路线判定系统中，已有"≥N个角色深层关系事件完成"的判定。BE 触发同理——检查特定角色的信任事件完成数量，而非好感度数值。
